## 🚀 一键备份方案

### 方案A：打包整个工作目录（推荐）

```bash
# 在原服务器执行
cd /mnt/workspace
tar -czvf dogdong_backup_$(date +%Y%m%d).tar.gz working/

# 下载备份文件，上传到新服务器
# 在新服务器执行
cd /mnt/workspace
tar -xzvf dogdong_backup_YYYYMMDD.tar.gz
```

### 方案B：分步备份

```bash
# 1. 备份核心身份文件
tar -czvf core_identity.tar.gz \
  working/AGENTS.md \
  working/MEMORY.md \
  working/PROFILE.md \
  working/SOUL.md \
  working/HEARTBEAT.md \
  working/BOOTSTRAP.md

# 2. 备份记忆目录
tar -czvf memory_backup.tar.gz working/memory/

# 3. 备份技能系统
tar -czvf skills_backup.tar.gz working/active_skills/

# 4. 备份脚本程序
tar -czvf scripts_backup.tar.gz working/scripts/

# 5. 备份定时任务配置
tar -czvf jobs_backup.tar.gz \
  working/jobs.json \
  working/task_registry.json \
  working/config.json

# 6. 备份数据文件
tar -czvf data_backup.tar.gz working/data/
```

## 📝 详细迁移步骤

### 步骤 1：备份原服务器

```bash
# SSH 登录原服务器
cd /mnt/workspace

# 创建完整备份
tar -czvf dogdong_full_backup_$(date +%Y%m%d_%H%M%S).tar.gz working/

# 查看备份大小
ls -lh dogdong_full_backup_*.tar.gz
```

### 步骤 2：传输备份文件

```bash
# 方法1: 使用 scp
scp dogdong_full_backup_*.tar.gz user@new-server:/mnt/workspace/

# 方法2: 使用 rsync
rsync -avz dogdong_full_backup_*.tar.gz user@new-server:/mnt/workspace/
```

### 步骤 3：新服务器解压

```bash
# SSH 登录新服务器
cd /mnt/workspace

# 解压备份
tar -xzvf dogdong_full_backup_*.tar.gz

# 验证文件完整性
ls -la working/
ls -la working/memory/
ls -la working/scripts/
ls -la working/active_skills/
```

### 步骤 4：设置环境变量

```bash

nano ~/.bashrc

export TOGETHER_API_KEY="tgp_v1_cBCC4a3n5xuZhdLBYGvCIacKmCm-X6vUebACCS0Nzeo"
```

### 步骤 5：安装 Python 依赖

## ⚠️ 重要注意事项

### 1. 飞书/钉钉 Webhook

如果使用飞书或钉钉推送，需要确认：
- Webhook URL 是否仍然有效
- session_id 和 user_id 是否正确

当前配置：
- 飞书 session_id: `6661ad16`
- 飞书 user_id: `董欣#ad16`

### 2. 定时任务

迁移后需要重新配置定时任务：
- 检查 `jobs.json` 中的 cron 表达式
- 确认 copaw 服务正在运行
- 验证任务是否按时触发

### 3. API 限流

注意各 API 的调用限制：
- Tavily: 免费版有月度限制
- Together AI: 按使用量计费
- StockAPI: 可能有频率限制

### 4. 文件权限

```bash
# 确保文件权限正确
chmod -R 755 /mnt/workspace/working/
chmod +x /mnt/workspace/working/scripts/*.py
```
