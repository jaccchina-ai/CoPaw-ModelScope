## 二、当前打分逻辑审视

### T日晚间选股评分（9个指标）

| 序号 | 指标 | 当前逻辑 | 权重 |
|------|------|---------|------|
| 1 | 首次涨停时间 | 10:00前=20分，10:30前=15分... | 15% |
| 2 | 封成比 | ≥10倍=15分，≥5倍=10分，≥3倍=5分 | 12% |
| 3 | 封单/流通市值 | ≥5%=15分，≥3%=10分，≥1%=5分 | 10% |
| 4 | 龙虎榜 | 有=10分，无=0分 | 10% |
| 5 | 主力资金净占比 | ≥10%=10分，≥5%=7分，≥0%=3分 | 12% |
| 6 | 成交金额 | 5-20亿=10分，2-50亿=5分 | 8% |
| 7 | 换手率 | 5-15%=10分，3-20%=5分 | 10% |
| 8 | 量比 | ≥5=10分，≥3=8分，≥2=5分 | 13% |
| 9 | 热点板块 | 按板块排名1-10分 | 10% |

### ⚠️ 发现的问题

#### 1. 缺失的重要指标

| 缺失指标 | 重要性 | API可用性 | 说明 |
|---------|--------|----------|------|
| **连板数** | ⭐⭐⭐⭐⭐ | ✅ ZTPool返回lbNum | 连板股成功率更高 |
| **炸板次数** | ⭐⭐⭐⭐ | ✅ ZTPool返回bombNum | 炸板多说明分歧大 |
| **板块强度** | ⭐⭐⭐⭐ | ✅ 热点板块API | 板块强度影响个股持续性 |
| **涨停原因** | ⭐⭐⭐⭐ | ✅ ZTPool返回plate_name | 涨停逻辑是否硬核 |
| **资金流入天数** | ⭐⭐⭐ | ✅ 热点板块API | 资金持续流入更可靠 |

#### 2. 当前逻辑缺陷

| 问题 | 现状 | 影响 |
|------|------|------|
| 龙虎榜评分过于简单 | 仅有/无，没有区分游资强度 | 无法区分章盟主 vs 散户 |
| 热点板块匹配粗糙 | 简单字符串匹配 | 可能漏配或错配 |
| 缺少连板判断 | 没有使用lbNum | 一板和高板同等对待 |
| 没有排除ST股 | 未过滤ST | 可能选中ST股 |
| 没有排除新股 | 未过滤新股 | 新股风险高 |

#### 新增指标

```python
# 1. 连板数评分（新增）
```

### 5.3 风控逻辑优化

```python
# 新增风控规则

def check_stock_risk(self, stock_data):
    """单只股票风控"""
    
    # 1. 连板高度风险
    lb_num = stock_data.get('lbNum', 1)
    if lb_num >= 5:
        return {'risk': 'high', 'reason': f'{lb_num}连板，位置过高'}
    
    # 2. 炸板风险
    bomb_num = stock_data.get('bombNum', 0)
    if bomb_num >= 2:
        return {'risk': 'medium', 'reason': f'炸板{bomb_num}次，分歧较大'}
    
    # 3. 板块过热风险
    if sector_strength > 100000:
        return {'risk': 'medium', 'reason': '板块过热，注意回调'}
    
    return {'risk': 'low', 'reason': '正常'}
```
